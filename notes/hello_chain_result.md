# 05.01 LangChain 与 LCEL · 实跑记录

- USE_CHAT: `True`
- topic: 什么是 RAG
- model: `astron-code-latest`

## STEP 1 · 逐节

- Prompt → `ChatPromptValue`
- Model → `AIMessage`
- Parser → str，前 200 字：RAG 就是检索增强生成，简单说就是给大模型外挂了一个知识库，让它回答问题前先去查资料。这样一来，大模型就能根据你提供的最新或专业内容来回答，既减少了胡说八道，又不用重新训练模型。

## STEP 2 · 整链

```text
RAG 就是检索增强生成，简单说就是让大模型在回答问题前，先去外部知识库里搜相关的资料。这样大模型就能根据搜到的真实内容来生成答案，而不是光凭自己脑子里的老记忆瞎编。这招既能让大模型获取最新知识，又能大大减少它胡说八道的情况。
```

## STEP 3 · 换 topic

```text
退货时效就是咱们买东西后，申请退货必须在规定的时间范围内操作，比如常见的“7天无理由”就是最典型的时效限制。如果超过了这个期限，系统通常就不允许你提交退货申请了。所以收到货后如果有不满意，一定要抓紧时间在有效期内处理，免得过期作废。
```

## 结论

- LangChain 属于专栏三层里的 Framework 层。
- LCEL：`prompt | model | parser`，统一 `invoke`。
- 调试时先逐节 invoke，再合成整链。
